# 1. Project Overview
Nexus is a professional, AI-first desktop IDE platform built with Electron, Angular, and TypeScript/Node.js. The goal is to deliver a plugin-friendly workbench on par with IntelliJ IDEA and VS Code while adding native local-AI capabilities powered by llama.cpp. The platform will expose a modular architecture with first-class extension APIs, high-performance workspace handling, and enterprise-grade security, observability, and packaging.

Key experience pillars:
- AI-native workflows (chat, inline code, agents) that work offline yet scale to future remote models.
- Extensible core featuring editor, explorer, SCM, terminal, search, run/debug/test, settings, themes, and marketplace-ready extension APIs.
- Resilient desktop shell optimized for multi-window, multi-root projects, and large repositories with robust telemetry and security.

# 2. Architecture Assumptions
- Electron main process orchestrates windows, IPC, auto-update, and OS integrations; preload bridges expose secure APIs to the Angular renderer.
- Angular provides the primary workbench UI; shared TypeScript packages host contracts, models, and state containers reused by Electron and Node services.
- Node-based background services (indexing, AI orchestration, extension host) run as isolated workers or processes communicating over typed IPC channels.
- llama.cpp runs as a managed local subprocess with a controller service for lifecycle, model selection, telemetry, and token streaming.
- User/workspace persistence paths resolve via env vars (`NEXUS_HOME`, `NEXUS_WORKSPACE_DATA`) that default to `.nexus` under each user profile, ensuring consistent storage across platforms.
- Plugin-first mindset: core features are built on the same APIs made available to third parties, ensuring future marketplace viability.
- Infrastructure is monorepo-based with shared tooling (e.g., Nx/Turborepo) for incremental builds, testing, and release.

# 3. Delivery Principles
- **AI-first but offline-capable**: prioritize flows that showcase on-device intelligence with guardrails and privacy.
- **Extension-before-feature**: design APIs and isolation before adding core features to guarantee long-term ecosystem viability.
- **Performance budgets**: enforce explicit budgets for startup (<3s cold on reference machine), workspace load (<5s for 100k files), and AI latency.
- **Incremental hardening**: deliver features in MVP form but pair with observability, feature flags, and fallback paths.
- **Security by construction**: minimize preload exposure, validate IPC contracts, and adopt an untrusted workspace model from the start.
- **Documentation + tests as artifacts**: every epic ships docs, sample usage, and automated tests in the same phase.
- **Enterprise readiness**: include telemetry, policy controls, offline mode, and packaging for Win/macOS/Linux before GA.
- **Localization-ready**: bake internationalization (i18n) infrastructure, locale switching, and translation workflows into core UI from day one to avoid costly retrofits.

# 7. Acceptance Criteria Per Epic
- **Platform Foundations (IDE-001–IDE-146, IDE-180)**: Fresh clone on macOS/Windows/Linux runs `yarn install && yarn nx run desktop-shell:serve` without manual tweaks; CI enforces ADR links for architectural PRs; distributed cache shows ≥40% time savings in build telemetry; devcontainer build parity verified weekly; env vars (`NEXUS_HOME`, `NEXUS_DATA_DIR`) default to `.nexus` and migrations log moves.
- **Electron Shell (IDE-009–IDE-016)**: Cold start telemetry ≤3s P95 on reference hardware; secure preload exposes only typed APIs; crash handler captures dumps and relaunches in safe mode; auto-update verifies signatures before prompting users.
- **Workbench Shell & Panels (IDE-017–IDE-025, IDE-147–IDE-148, IDE-181–IDE-182)**: Activity bar, command palette, dockable panels, theming, and locale switching all operable via keyboard; problems/output panels handle 5k entries while maintaining <16ms frame time; layout state persists per workspace; untranslated strings fail CI and runtime falls back cleanly.
- **Icon System Platform (IDE-190–IDE-197)**: Registry resolves codicon/file/folder/theme/custom icons with <100µs lookup time, theme switches update icon classes globally in <100ms, built-in pack covers ≥90% of tracked file/folder types (validated via test fixtures), Angular `IconComponent` and `FileIconDirective` power workbench/explorer without duplicating logic, and telemetry/snapshot tests guard against regressions; extension manifest can declare icons once IDE-195 lands.
- **Workspace System (IDE-026–IDE-033, IDE-149–IDE-150)**: Multi-root workspaces persist MRU + session state; 100k-file repo monitoring delivers FS events <1s; crash-restore recovers dirty buffers and terminals; remote/virtual FS providers permissioned and auditable.
- **File Explorer (IDE-034–IDE-040, IDE-150)**: CRUD/drag/drop propagate to disk with optimistic updates + rollback; Git/problem decorations refresh <500ms post-change; virtual FS contributions obey capability prompts and survive reconnect.
- **Editor Platform (IDE-041–IDE-047, IDE-151–IDE-153)**: Files open/save/autosave reliably; formatting on save respects workspace profiles; navigation (goto/rename) completes <250ms; CodeLens and diagnostics integrate with problems panel; large-file mode keeps UI responsive (<400ms).

- **SCM/Git (IDE-054–IDE-059, IDE-154–IDE-155)**: Repo discovery handles mono/multi-root; staging/diff/commit/his­tory operate without shelling out; branch switch restores workspace context; blame/stash/cherry-pick UIs function with AI-assisted messages available.

- **Run/Debug/Test (IDE-064–IDE-069, IDE-157–IDE-158)**: Launch configurations validate schema with helpful errors; Node + Python adapters run via built-in host; breakpoints/watch expressions persist; test explorer executes Jest/Vitest suites, surfaces coverage overlays.
